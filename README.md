# lidar-dynamic-obstacles

**Детекция и сопровождение динамических препятствий в реальном времени по данным лидара Hokuyo: DBSCAN-кластеризация + фильтр Калмана + вычитание статического фона.**

## Описание

Конвейер обработки лидарных данных, который в реальном времени (40 Гц) выделяет в окружении движущиеся объекты, отличает их от статического фона и устойчиво сопровождает фильтром Калмана. Работает напрямую с реальным лидаром Hokuyo UTM-30LX-EW по Ethernet (протокол SCIP 2.2). Тестировался на Raspberry Pi Zero W: устойчивое сопровождение объектов со скоростью до 2 м/с, задержка классификации «статика/динамика» ~200 мс.

## Алгоритм

Каждый кадр проходит через конвейер:

1. **Декодирование SCIP 2.2 → облако точек.** Сырой ответ лидара (3-символьное ASCII-кодирование расстояний) разбирается в массив дальностей, который переводится в декартовы координаты.

2. **Кластеризация (DBSCAN-подобная).** Точки группируются по близости: разрыв между соседними точками больше порога → начинается новый кластер. Так из сплошного облака выделяются отдельные объекты.

3. **Геометрическая фильтрация кластеров.** Отбрасываются кластеры неподходящего размера: слишком мелкие (шум) и слишком крупные/вытянутые (стены, пол). Критерии — размер bounding box по осям и соотношение сторон. Зачем: оставить только объекты «человеческого» масштаба, убрав заведомую статику.

4. **Вычитание статического фона.** Поддерживается occupancy grid с экспоненциальным скользящим средним (коэффициент забывания α): ячейки, стабильно занятые из кадра в кадр, считаются фоном (порог θ) и исключаются. Это убирает неподвижные стены/мебель ещё до трекинга.

5. **Трекинг фильтром Калмана.** Каждый объект ведётся КФ с вектором состояния «координаты + скорость». Ассоциация наблюдений с треками — по дистанции (метрика близости центроидов). КФ сглаживает измерения и даёт оценку скорости, по которой выносится вердикт «статический / динамический».

6. **Подавление ложных кластеров в тени лидара.** За реальным объектом лидар «не видит» — там возникают артефакты. Два независимых правила убирают такие ложные треки: (1) угловой конус позади объекта с запасом Δ; (2) приоритет ближайшего объекта в зоне `d_зона`. Зачем: не плодить фантомные препятствия из окклюзии.

7. **Компенсация движения платформы.** Если робот сам движется, одометрия переводится в матрицу жёсткого преобразования T, которая дополнительно уточняется ICP по точкам статического фона. Так движение объектов отделяется от собственного движения сенсора.

## Технический стек

| Категория   | Компоненты |
|-------------|------------|
| Вычислитель | Raspberry Pi Zero W (и выше) |
| Лидар       | Hokuyo UTM-30LX-EW, SCIP 2.2, 1080 точек/скан, FOV 270°, 40 Гц |
| Связь       | Ethernet / TCP, порт 10940 |
| Алгоритмы   | DBSCAN-кластеризация, фильтр Калмана 2D, occupancy grid (EMA), ICP |
| Библиотеки  | NumPy, OpenCV (визуализация); `socket`, `math`, `collections` — stdlib |

| `requirements.txt` | Зависимости Python |

## Как запустить

1. **Установить зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

2. **Подключить лидар** по Ethernet и убедиться, что он доступен по адресу `192.168.0.10:10940` (при необходимости поправьте `LIDAR_IP` / `LIDAR_PORT` в начале файла).

3. **Запустить:**
   ```bash
   python dynamic_obstacles.py
   ```

4. **Управление в окне:**
   - **Q** — выход;
   - **R** — сбросить все треки;
   - **+ / −** — масштаб;
   - **SPACE** — пауза.

## Параметры

Настройки — в начале файла `dynamic_obstacles.py` (блок «НАСТРОЙКИ»).

| Параметр | Значение | Назначение |
|----------|----------|------------|
| `NUM_POINTS` | 1080 | Точек в скане |
| `DIST_MIN_MM` / `DIST_MAX_MM` | 100 / 15000 | Валидный диапазон дальности, мм |
| `FRAME_DT` | 0.05 с | Интервал между кадрами |
| `CLUSTER_SPLIT_MM` | 200 мм | Порог разрыва для разбиения на кластеры (ε) |
| `CLUSTER_MIN_PTS` / `MAX_PTS` | 25 / 200 | Мин./макс. число точек в кластере (MinPts) |
| `CLUSTER_MIN_SIZE_MM` / `MAX_SIZE_MM` | 50 / 1500 | Допустимый размер кластера, мм |
| `CLUSTER_MAX_ASPECT` | 6.0 | Макс. соотношение сторон (отсечь стены) |
| `MATCH_DIST_MM` | 600 мм | Радиус ассоциации кластера с треком |
| `MAX_LOST_FRAMES` | 8 | Кадров до удаления потерянного трека |
| `MIN_AGE_DYNAMIC` | 8 | Кадров наблюдения до вердикта «динамический» |
| `DYNAMIC_SPEED_MM_S` | 250 мм/с | Порог скорости «статика/динамика» |
| `SHADOW_MARGIN_DEG` | 30° | Запас углового конуса (правило 1 теней) |
| `SHADOW_ZONE_MM` / `SHADOW_ZONE_DEG` | 3000 мм / 20° | Зона и угол правила 2 подавления теней |

## Известные ограничения

- Стеклянные/зеркальные поверхности дают ложные дальности → ложные или пропущенные объекты.
- Очень близко стоящие друг к другу объекты могут слиться в один кластер.
- Объект, надолго заслонённый другим (полная окклюзия дольше `MAX_LOST_FRAMES`), теряется и заводится как новый трек.
- Компенсация собственного движения опирается на качество одометрии и наличие статического фона для ICP; на открытом пространстве без стен уточнение хуже.
- Очень быстрые объекты (>2 м/с) могут «выпадать» из радиуса ассоциации между кадрами.
